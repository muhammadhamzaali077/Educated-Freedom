_Generated by `pnpm tsx scripts/canva-diagnostic.ts`. The script deliberately calls the Canva token endpoint with an invalid authorization code to exercise the diagnostic logging path without requiring an interactive OAuth round-trip._

## Response captured by `CanvaApiError`

- **status**: 400

- **body** (first 1000 chars):

```json
{"error":"invalid_grant","error_description":"Invalid auth code"}
```

## What this confirms

- Diagnostic logs (request → headers, body kind; response ← status, content-type, body) print under the `[canva:requestId]` prefix.
- `Authorization: Basic …` is scrubbed to `Basic ***` before logging.
- The error path throws `CanvaApiError(status, bodyText, url, method)` with the actual Canva message intact.
- The route handler catches `CanvaApiError` and returns `Canva {status}: {bodyText}` to the client, where `canva-fallback.js` handles the inline-message + auto-PDF + new-tab fallback.
